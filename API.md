# GymOrNot API Reference

This document describes the API routes used by GymOrNot, their contracts, and how they integrate with the quiz flow.

## API route overview

GymOrNot exposes server-side API routes for quiz data and evaluation:

- `GET /api/quiz-data` — Fetches quiz questions
- `POST /api/quiz-data` — Scores user answers and returns the archetype diagnosis

Both routes are implemented in `app/api/quiz-data/route.ts`.

---

## 1. GET /api/quiz-data

### Purpose
Retrieves the 4 questions parsed from the questions database or generated by AI. It returns options configured with a 4-axis scoring model.

### Request
Standard `GET` request. No body or parameters required.
```js
fetch('/api/quiz-data')
```

### Response
Returns a JSON object containing the questions:
```json
{
  "questions": [
    {
      "id": "q1",
      "eyebrow": "The Track Record",
      "prompt": "Be honest with your maker. The last time you had a fitness-adjacent goal, what happened?",
      "options": [
        {
          "label": "I went 4 times a week, tracking every macro like a scientist.",
          "gymScore": 2,
          "homeScore": 1,
          "boutiqueScore": 1,
          "couchScore": 0
        }
      ]
    }
  ]
}
```

---

## 2. POST /api/quiz-data

### Purpose
Calculates the user's score across all 4 axes, derives their risk archetype, and computes their dropoff probability.

### Request
`POST` request with the user's answers:
```json
{
  "answers": [
    {
      "id": "q1",
      "optionIndex": 0
    }
  ]
}
```

### Response
Returns the tallied scores, the computed archetype, and dropoff probability:
```json
{
  "gymScore": 2,
  "homeScore": 1,
  "boutiqueScore": 1,
  "couchScore": 0,
  "archetype": {
    "id": "gym-crusader",
    "name": "The Gym Crusader",
    "roast": "You are the person the ghost-members are subsidizing...",
    "shareHeadline": "Verified 'Gym Crusader.'...",
    "color": "green",
    "cta": {
      "label": "You belong in a gym...",
      "href": "#gyms-placeholder",
      "type": "leadgen"
    }
  },
  "dropoffProbability": 0,
  "perQuestion": [
    {
      "id": "q1",
      "selected": "I went 4 times a week, tracking every macro like a scientist."
    }
  ]
}
```

## Env vars and AI integration

The route supports optional AI-backed generation through two environment variables:
- `GEN_AI_ENDPOINT`
- `GEN_AI_API_KEY`

If both variables are available, the route attempts to fetch questions from the external AI endpoint. If the request fails, or if variables are missing, it falls back to the built-in questions in `lib/quiz.ts`.
