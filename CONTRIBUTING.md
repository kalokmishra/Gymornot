Thank you for wanting to contribute to GymOrNot! We welcome bug reports, feature requests, documentation fixes, and code changes.

Getting started
- Fork the repository and create a feature branch from `main`.
- Install dependencies and run locally:

```bash
npm install
npm run dev
```

Branching & PRs
- Use descriptive branch names: `feature/short-description` or `fix/short-description`.
- Open a pull request against `main` with a clear title and description. Link related issues.
- Keep PRs small and focused when possible.

Code style & linting
- This project uses TypeScript, Tailwind CSS and Next.js. Please follow existing code patterns.
- Run the linter before committing:

```bash
npm run lint
```

Testing
- There are no automated tests set up yet. If you add tests, include instructions in the PR description on how to run them.

Commit messages
- Use clear, imperative commit messages. Prefer Conventional Commits (e.g. `feat:`, `fix:`) to make the history easier to read.

Reporting issues
- Use the issue templates when opening bug reports or feature requests. Provide reproduction steps and expected behaviour.

Security
- For security issues, do not open a public issue. Contact the repository owner directly or use a private channel.

Thanks — we look forward to your contributions!
